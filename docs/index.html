What is this?

Every hour, roughly 25 NOAA weather stations across Southern California report temperature, pressure, humidity, wind speed, and dewpoint. Conventional weather analysis looks at these numbers station by station: "Is LAX getting warmer? Is pressure dropping at San Diego?"

This system does something different. It ignores the individual readings and instead analyzes the relationships between stations — how does temperature at Santa Barbara differ from Van Nuys? When pressure diverges between two stations, does humidity also diverge? It builds a mathematical picture of how the entire network of stations is connected.

The key finding: when a weather system is approaching (a front, marine surge, Santa Ana event), the pattern of relationships across the station network begins to reorganize before any individual station's readings show obvious changes. The system detects this reorganization and makes a timestamped, public prediction.

The claim being tested: This mathematical analysis of station relationships detects approaching weather changes approximately 4–10 hours before conventional instruments show them. Every prediction is publicly logged and automatically verified. This dashboard is the scoreboard.

How does the math work?

Three mathematical operators (A, B, R) are applied to the station network each hour. Each one builds on the previous:

  • A Measure differences — For every pair of nearby stations, compute how their readings differ. "Santa Barbara is 5°F warmer than Van Nuys. LAX has 2 mb more pressure than Ontario." These pairwise differences are the raw signal.
  • B Track patterns along chains — A single station pair might show a random fluctuation. But when differences build up consistently along a chain of stations from coast to inland, that indicates a real spatial pattern. B accumulates differences along these chains.
  • R Detect cross-coupling — This is the key step. R asks: when the temperature gradient between two stations is large, does the pressure gradient respond too? Does humidity couple with wind? When weather variables start moving together across the network, something is coming.
  • Γ The output number — Γ (Gamma) is a single number summarizing how much cross-coupling the whole network shows right now. ΔΓ (Delta Gamma) is how fast Γ is changing. A spike in ΔΓ means the network's relational structure is reorganizing rapidly.
When the change index (ΔΓ) spikes above its statistical threshold, the system predicts that conventional weather readings across the network will show significant coordinated changes within 4–10 hours. The chart below shows this index in real time.

Technical details

Raw ΔΓ value: +668 (before normalization)

The "relationship change index" shown on this dashboard is the normalized ΔΓ: the raw change divided by the current Γ magnitude. This produces a dimensionless ratio that is comparable across different network sizes and conditions. The raw value is in arbitrary units determined by station geometry.

Detection parameters:

  • Threshold: 1.5 standard deviations above rolling baseline
  • Operator variant: normalized
  • Prediction window: 4–10 hours after detection
  • Debounce: 6-hour cooldown between predictions (prevents clustering)
  • Topology guard: detection suppressed if more than 2 stations drop/join (avoids artifacts from network reconfiguration)

Calibration status: BORDERLINE

Mathematical basis: V4 ABR operators (A: gradient, B: accumulation, R: circulation) applied over declared station-pair relations with 5 weather components (temperature, pressure, humidity, wind speed, dewpoint depression).
